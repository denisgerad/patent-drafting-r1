Good call — patent drafts are highly sensitive. The right approach here is a local Python app that runs entirely on your machine. Since you're already using Mistral 7B + CrewAI, you likely have Ollama running. Here's exactly how to set it up:

Single file, runs entirely offline. Here's how to get it going:

Step 1 — make sure Ollama is running

bash
ollama serve
ollama pull mistral   # or llama3, phi3, etc.
Step 2 — install the two Python deps

bash
pip install flask requests
Step 3 — run it

bash
python patent_analyzer.py
Then open http://localhost:5000 in your browser.

What you get:

The UI is a dark terminal-style dashboard. Paste your draft, hit Analyze, and in ~30–60 seconds (depending on your hardware) you'll see:

Dimension	What it checks
Completeness	Are all patent sections present?
Technical depth	Specific numbers, protocols, algorithms vs. vague prose
Claimability	Are there defensible novel elements?
Ambiguity clarity	How precise and unambiguous is the language?
Hallucination risk	Are all assertions grounded / verifiable?
Implementation detail	Can an engineer build from this description alone?
Terminology quality	Correct patent language throughout?
Plus an overall score, a Ready / Needs clarification / Poor quality verdict, and the top 3 issues to fix before handing off for domain review.

To switch models, just change line 22:

python
OLLAMA_MODEL = "llama3"   # or "phi3", "codellama", "gemma", etc.
Larger models (13B+) will give more reliable JSON output than 7B.
